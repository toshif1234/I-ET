Admin Console

User Entry Page:

The user entry page allows the admin to add/delete users as well as toggle user access to the app and define specific roles.

  • Status: It is the ability to enable or disable a user's access to the app. The admin can toggle the status on or off using a checkbox to either grant or restrict the user's access to the app.
  • Filtering by Status: To filter users by status, use the search or filter options provided in the admin console. Use 1 to filter active users and use 0 to filter inactive users from the database.
  • Roles: Roles determine the level of access and features available to a user within the application. Each role is designed to suit different responsibilities and access needs.

Roles:

The Roles defined in StockSnap are as follows:

  • Admin Role: A super-user role that provides access to all features of the application. Admins can manage all aspects of the system, including user management and version control.
  • Manager Role: Has access to most features of the admin console but is restricted to specific warehouses. The manager cannot add new users or manage version control of the app.
  • Operator Role: Can use the handheld terminal for entering data into the application. Operators do not have access to the admin console or management features.

Customization

The Customizing Screen allows the admin/manager to configure settings for the handheld terminal (HHT). These tools help tailor the HHT interface without compromising data integrity.

Unit of Measure (UoM):

Enter multiple units of measure (UoMs) for the operator to use, separated by commas. Example Template: EA, CV, PAL.

Field Toggles:

  • Field Toggles: Use checkboxes to select which fields should appear on the HHT for the operator. Toggle them on or off to customize visibility.
  • Field Names: Field names can be edited to personalize the HHT screen for your business. Ensure the nature of the fields remains unchanged. For instance: Do not write "Product" in the place of the "Bin" field or vice versa.

Import Master Data

The Import Master Data Screen allows users to upload master data into the application. There are two ways to upload master data:

Using the Template:

  • Download the provided template.
  • Fill in the required fields in the template file.
  • Upload the completed template back into the system.

Updating SAP Export:

  • Export the LS11 bin or SE16 bin master data from SAP.
  • Update the following field labels in the exported file to match the following format: Storage_Bin, Storage_Type, Storage_Section.
  • Save and upload the updated file.

Important Notes: If duplicate bins are found in the uploaded data, none of the bins will be uploaded. Correct the duplicates and re-upload the file. This functionality ensures a seamless and accurate import of master data.

Manage Entries

The Manage Entries Screen allows the manager to review, approve, and update bin entries. This screen simplifies managing bin entries while ensuring accuracy and efficiency in the approval process.

Approval Process:

Admin/Managers have the ability to approve which entries are ready to be exported adding another layer of manual data verification. The steps to approve are as follows:

  • Select an entry/entries that you want to export and click the approve button.
  • Approved entries are flagged in green color.

Note: Bins can only be approved once. After approval, no further changes can be made to the bin entry.

Legend:

Entries are marked with colored flags to indicate their status:

  • Approved: Indicates the bin has been reviewed and approved.
  • Not Approved: Indicates the bin is pending review or requires changes.
